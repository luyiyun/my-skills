# 每日英语学习 Skill

## 核心功能

帮助用户进行结构化的每日英语学习（5-10分钟），管理学习进度和个人偏好，并将学习内容保存到 Obsidian。

## 文件夹结构

英语学习文件的组织结构（直接保存在当前工作目录下）：

```
./
├── 📋 english-learning-profile.md  # 核心记忆文件（必须首先读取/创建）
├── 📁 by-topic/                    # 按主题分类汇总
│   ├── vocabulary.md
│   ├── expressions.md
│   ├── grammar.md
│   └── reading.md
└── 📁 by-date/                     # 按日期归档
    ├── 2025-03/
    │   ├── 2025-03-21.md
    │   └── 2025-03-22.md
    └── 2025-04/
        └── ...
```

## 学习流程

### 阶段 1: 准备 - 读取记忆文件

**必须首先执行：**

1. **立即在当前工作目录下查找 `english-learning-profile.md`**（不要先问用户）：
   - **如果找到**：读取该文件，获取当前主题序号、学习进度、个人偏好，直接进入阶段 2
   - **如果没找到**：说明当前目录可能不对，或者需要新建档案。此时询问用户：
     > "在当前目录下没有找到 `english-learning-profile.md` 学习档案。请问：
     > 1. 是否需要切换到正确的学习文件夹？（如果是，请告诉我路径）
     > 2. 还是要在当前目录新建一个学习档案？"
   - 根据用户回答：
     - 如果切换目录：在用户指定路径下重新查找 `english-learning-profile.md`
     - 如果新建档案：询问基本信息（当前水平、学习目标、感兴趣话题），然后创建 `english-learning-profile.md`

2. **读取档案，提取关键状态：**
   - `current-topic-index`：确定今日学习主题
   - `last-updated`：判断上次学习日期，用于计算 streak 并生成开场问候
   - `interested-topics`、`weak-points`、`preferred-difficulty`：用于定制今日内容

3. 根据 `current_topic_index` 确定今日主题：
   - 0 → 词汇积累
   - 1 → 地道表达
   - 2 → 语法精讲
   - 3 → 阅读理解

### 阶段 2: 进行学习

#### 开场问候
参考记忆文件中的 `last-learning-date`，如果：
- 昨天学过 → "欢迎回来！这是你的第 X 天学习 🔥"
- 断了一天 → "好久不见！让我们继续学习之旅"
- 第一次 → "欢迎开始你的英语学习之旅！"

#### 根据今日主题提供内容

##### 1. 词汇积累日
- 提供 3-5 个高频词汇
- 从 `learning-profile.md` 中 `interested-topics` 选择相关主题
- 每个词：英文释义、中文释义、2 个例句、常见搭配
- **记录到 `by-topic/vocabulary.md` 和 `by-date/YYYY-MM/YYYY-MM-DD.md`**

##### 2. 地道表达日
- 提供 2-3 个实用短语/习语
- 每个：字面意思、实际含义、使用场景、例句、正式程度
- **记录到 `by-topic/expressions.md` 和日期文件**

##### 3. 语法精讲日
- 聚焦一个具体语法点
- 优先选择 `weak-points` 中记录的薄弱点
- 包含：规则说明、例句、常见错误及纠正
- **记录到 `by-topic/grammar.md` 和日期文件**

##### 4. 阅读理解日
- 提供 150-200 词短文
- 文章类型：故事、新闻、生活场景、科普
- 配 2-3 道理解题 + 答案解析
- 标出生词和重点表达
- **记录到 `by-topic/reading.md` 和日期文件**

#### 互动环节
- 根据今日主题出题让用户练习
- 给予积极反馈

### 阶段 3: 保存 - 更新文件

**学习结束后，必须执行：**

1. **更新 `english-learning-profile.md`：**
   - `last-updated`: 今天日期
   - `total-sessions`: +1
   - `current-topic-index`: (当前值 + 1) % 4
   - 对应主题计数 +1
   - 更新 streak 记录
   - 添加今日学习的主题到已学习列表（避免重复）

2. **创建/更新 `by-date/YYYY-MM/YYYY-MM-DD.md`：**
   - 使用日期文件模板

3. **追加到对应的 `by-topic/*.md`：**
   - 将今日内容添加到相应主题文件

4. **询问用户：**
   - "今天要更新学习档案吗？（比如添加新的薄弱点、调整偏好）"
   - 如果用户有反馈，更新 `english-learning-profile.md`

## 特殊情况处理

| 情况 | 处理方式 |
|------|----------|
| 用户指定特定主题 | 尊重选择，但不改变轮换序号，下次继续原计划 |
| 用户想复习之前的内容 | 读取对应的日期文件或主题汇总文件 |
| 用户想跳过今天 | 记录到 profile，streak 中断，下次继续 |
| 用户想调整难度 | 更新 `learning-profile.md` 中的偏好 |
| 用户不想保存到 Obsidian | 尊重选择，仅显示学习内容，但建议至少创建 profile 文件 |
| 文件夹不存在 | 询问用户是否创建 |

## 更新学习档案的时机

1. **每次学习结束时**（自动更新进度统计）
2. **用户主动要求时**（如"更新我的学习目标"）
3. **发现新的薄弱点时**（用户犯错或表达困难）
4. **完成阶段性目标时**（如"已学完基础词汇"）
